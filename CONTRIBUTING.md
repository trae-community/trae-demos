# Contribution Guidelines

Welcome to contribute to TRAE Demos! We appreciate your submissions and help in building a vibrant community of projects built with TRAE.

![TRAE Demos Banner](./assets/image/Demos.gif)

## 📋 What We're Looking For

We accept projects that demonstrate meaningful use of TRAE in development. Your project should:

### ✅ Must Have
- **Uses TRAE**: TRAE must be a core technology in your development process
- **Accessible**: Public repository or live demo available
- **Quality**: Good code quality with basic documentation
- **Complete**: High level of polish and functionality

### 🎯 Nice to Have
- Clear demonstration of TRAE features used
- Innovative technical implementations
- Well-structured codebase
- Comprehensive documentation
- Live demo or deployment

## 🚀 How to Submit a Project

### Step 1: Prepare Your Project

Before submitting, ensure:
- Your project is in a public repository
- You have a README with basic information
- Your project is functional and well-tested
- You have screenshots or demos ready

### Step 2: Create an Issue

1. Go to [Issues](https://github.com/trae-community/trae-demos/issues)
2. Click "New Issue"
3. Select the appropriate template
4. Fill in all required fields

### Step 3: Fill in the Template

Provide complete information:

#### Project Information
- **Project Name**: Clear, descriptive name
- **Repository URL**: Link to your public repo
- **Live Demo URL**: If available (highly encouraged)
- **Project Type**: Choose the most appropriate category

#### Description
- **One-line Description**: Elevator pitch (under 140 characters)
- **Core Features**: 3-5 standout features
- **Technical Highlights**: Explain how TRAE was used and any innovations

#### Screenshots
- Provide 2-3 clear screenshots or GIFs
- Show key features or UI
- Include brief captions

### Step 4: Review and Submit

Before clicking submit:
- ✅ All fields are filled out
- ✅ Links are working
- ✅ Screenshots are clear
- ✅ Description is compelling

### Step 5: After Submission

1. **Acknowledgment**: You'll receive acknowledgment within 24 hours
2. **Review**: Our team will review within 3-5 business days
3. **Feedback**: We may ask for clarifications if needed
4. **Approval**: Once approved, your project will be featured!

## 📝 Project Categories

Choose the most appropriate category for your project:

### Web Applications
Full-stack web applications built with TRAE
- Examples: E-commerce sites, social platforms, dashboards
- Tech: React, Vue, Angular, Node.js, Python, etc.

### Tools & Utilities
Developer tools, CLI applications, productivity utilities
- Examples: Code generators, automation scripts, dev tools
- Tech: Any programming language

### Games
Interactive games and entertainment applications
- Examples: Browser games, puzzle games, interactive experiences
- Tech: Game frameworks, graphics libraries

### AI Applications
Projects leveraging AI capabilities with TRAE
- Examples: AI-powered apps, ML models, chatbots
- Tech: TensorFlow, PyTorch, AI APIs

### Other
Anything else that doesn't fit above categories
- Examples: Mobile apps, IoT projects, creative coding

## 🎯 Quality Standards

### Code Quality
- Clean, readable code
- Proper project structure
- No sensitive information (API keys, passwords)
- Appropriate use of version control

### Documentation
- Clear README with setup instructions
- Usage examples
- License information
- Contact or support information

### TRAE Usage
- Clear description of how TRAE was used
- Specific features or workflows
- Benefits gained from using TRAE
- Lessons learned or insights

## 🔍 Review Process

### Timeline
- **Day 1**: Acknowledgment received
- **Days 2-5**: Initial review and evaluation
- **Days 6-10**: Feedback and clarification (if needed)
- **Day 10+**: Final decision and featuring

### Review Criteria
Projects are evaluated on:
1. **TRAE Usage** (40%): How effectively TRAE was used
2. **Code Quality** (25%): Technical implementation
3. **Completeness** (20%): Level of polish and functionality
4. **Documentation** (15%): Clarity and presentation

### Possible Outcomes
- ✅ **Approved**: Project featured in demos
- ⚠️ **Needs Work**: Minor changes requested
- ❌ **Declined**: Doesn't meet requirements (with feedback)

## 💡 Tips for Success

### Do's
- Provide detailed information about TRAE usage
- Include working demos or clear screenshots
- Write compelling descriptions
- Highlight what makes your project unique
- Respond promptly to feedback

### Don'ts
- Submit incomplete information
- Include private/sensitive data
- Exaggerate TRAE usage
- Submit broken or unfinished projects
- Ignore feedback requests

## 🤔 Common Questions

### Q: Can I submit multiple projects?
A: Absolutely! Each project should be submitted as a separate issue.

### Q: Does my project need to be complete?
A: It should be functional and polished. We prefer showcasing completed or near-completed projects.

### Q: What if my project uses other AI tools too?
A: That's fine! Just clearly describe how TRAE specifically contributed to your development.

### Q: Who owns the copyright?
A: You retain all rights to your project. We just showcase it.

Your submission helps build a stronger community and inspires other developers. Every project contributes to our collective knowledge!
